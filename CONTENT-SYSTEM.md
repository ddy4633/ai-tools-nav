# 好工具 - 实时内容采集系统
## 自动化情报流水线设计

**核心目标**: 24小时内发现新工具，48小时内发布评测  
**价值主张**: 做「AI工具的TechCrunch」，不是「工具黄页」

---

## 一、情报来源监控

### 1.1 监控渠道

| 渠道 | 监控内容 | 频率 | 优先级 |
|------|----------|------|--------|
| **Product Hunt** | 每日热门、AI分类新品 | 每小时 | P0 |
| **GitHub Trending** | AI/ML相关仓库 | 每天 | P0 |
| **Hacker News** | Show HN、AI相关讨论 | 实时 | P1 |
| **Twitter/X** | AI工具发布、KOL推荐 | 实时 | P1 |
| **Reddit** | r/MachineLearning, r/LocalLLaMA | 每天 | P2 |
| **Discord** | 各大AI工具社区 | 每天 | P2 |
| **YC Batch** | 最新批次AI公司 | 每批次 | P1 |
| **独立开发者社区** | Indie Hackers, 即刻 | 每天 | P2 |

### 1.2 监控指标

- **热度指标**: 点赞数、评论数、下载量增速
- **质量指标**: 是否有实际产品（不只是Demo）
- **差异化指标**: 与已有工具的区别度
- **时效性指标**: 发布时间（24小时内为最佳）

## 二、自动化采集流水线

│ (API/爬虫)   │    │ (热度/质量)  │    │ (AI写作)     │    │ (快速校验)   │
└─────────────┘    └─────────────┘    └─────────────┘    └──────┬──────┘
                                                                  │
┌─────────────────────────────────────────────────────────────────┘
│
▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  自动发布    │───→│  社交媒体    │───→│  用户反馈    │

### 2.2 各阶段详情

#### Stage 1: 监控采集 (自动化)

**Product Hunt 监控**:
```python
# 每小时抓取 AI 分类的新品
# 筛选条件:
# - 发布 24 小时内
# - 点赞 > 50
# - 有实际产品链接（不只是落地页）
```

**GitHub Trending 监控**:
```python
# 每天抓取 AI/ML 相关仓库
# 筛选条件:
# - Star 增速 > 100/天
# - 有 README 和使用说明
# - 不是教程/列表类仓库
```

**Twitter/X 监控**:
```python
# 监控关键词: "launched", "shipping", "new AI tool"
# 关注列表: AI 领域 KOL、投资人、产品经理
# 筛选条件:
# - 转发 > 20
# - 有产品链接
```

#### Stage 2: 智能筛选 (自动化)

**初筛算法**:
```
得分 = 热度分 * 0.4 + 质量分 * 0.4 + 时效分 * 0.2

热度分 = min(点赞数/100, 10) + min(评论数/20, 5)
质量分 = 有产品演示? 5:0 + 有 pricing? 3:0 + 有文档? 2:0
时效分 = 24小时内发布? 10: 48小时内? 5: 1

通过阈值: 得分 >= 15
```

**去重机制**:
- 基于域名去重（避免同一产品多次采集）
- 基于功能相似度去重（避免功能雷同的工具）

#### Stage 3: AI内容生成 (自动化)

**生成流程**:
1. **深度研究**: 访问官网、试用产品、收集用户评价
2. **多维分析**: 功能、定价、竞品、适用人群
3. **内容生成**: 按模板生成评测
4. **事实核查**: 交叉验证关键信息

**生成模板** (同前，但增加实时性标记):
```markdown
---
status: draft
source: Product Hunt
discovered_at: 2026-02-17T10:00:00Z
published_at: null
viral_score: 23
---

# {工具名称}

🆕 **新品** | 发现于 {平台} {时间} ago

## 一句话
{核心卖点}

## 快速评测
{AI生成的300字评测}

## 实测截图
{自动抓取的产品截图}

## 社区反馈
- Product Hunt: {点赞数}👍 {评论数}💬
- Twitter: {转发数}🔄

## 是否值得跟进
{AI判断: 值得深度评测 / 观望}
```

#### Stage 4: 人工审核 (10分钟/篇)

**审核清单**:
- [ ] 产品真实存在（不是落地页）
- [ ] 核心功能描述准确
- [ ] 定价信息正确
- [ ] 无重大事实错误
- [ ] 语气符合品牌调性

**审核工具**:
- 一键发布到网站
- 一键发布到社交媒体
- 一键标记「需要深度评测」

#### Stage 5: 自动发布

**发布渠道**:
1. **网站**: 自动更新数据库，加入「最新发现」板块
2. **Twitter**: 自动生成推文（产品名+一句话推荐+链接）
3. **即刻**: 同步发布到即刻圈子
4. **Newsletter**: 每周汇总发送

## 三、技术实现

### 3.1 技术栈

```yaml
监控层:
  - Product Hunt API
  - GitHub API
  - Twitter API v2
  - RSS 订阅 (Hacker News, Reddit)
  
数据处理:
  - Python + FastAPI (采集服务)
  - PostgreSQL (数据存储)
  - Redis (缓存/队列)
  
AI生成:
  - 多模型协作 (GLM5+Kimi+MiniMax)
  - 自动事实核查 (交叉验证)
  
发布:
  - Supabase (网站数据)
  - Twitter API (社交媒体)
  - Telegram Bot (通知)
```

### 3.2 部署架构

```yaml
# cron-job.yaml
jobs:
  product-hunt-monitor:
    schedule: "0 * * * *"  # 每小时
    command: "python scripts/monitor_product_hunt.py"
    
  github-trending-monitor:
    schedule: "0 9 * * *"  # 每天早9点
    command: "python scripts/monitor_github.py"
    
  twitter-monitor:
    schedule: "*/30 * * * *"  # 每30分钟
    command: "python scripts/monitor_twitter.py"
    
  content-generator:
    schedule: "0 */6 * * *"  # 每6小时
    command: "python scripts/generate_content.py"
```

### 3.3 数据模型

```typescript
// Tool 模型扩展
interface Tool {
  // 基础信息
  id: string;
  name: string;
  
  // 实时性追踪
  discoveredAt: string;      // 发现时间
  publishedAt: string;       // 发布时间
  source: 'product-hunt' | 'github' | 'twitter' | 'hackernews' | 'manual';
  sourceUrl: string;         // 原始链接
  viralScore: number;        // 热度得分
  
  // 内容状态
  status: 'discovered' | 'generating' | 'pending-review' | 'published' | 'rejected';
  
  // 社区反馈（自动采集）
  socialMetrics: {
    productHunt?: { votes: number; comments: number };
    github?: { stars: number; forks: number };
    twitter?: { likes: number; retweets: number };
  };
  
  // 审核信息
  reviewedBy?: string;
  reviewedAt?: string;
  reviewNotes?: string;
  
  // 常规字段...
}
```

## 四、运营策略

### 4.1 内容分级

| 级别 | 标准 | 处理方式 |
|------|------|----------|
| **S级** | viral_score > 30，24小时内 | 立即审核，优先发布，全渠道推广 |
| **A级** | viral_score > 20，48小时内 | 当天审核发布 |
| **B级** | viral_score > 15，一周内 | 批量审核，定时发布 |
| **C级** | 其他 | 入库待观察 |

### 4.2 发布节奏

- **实时**: Twitter/X 立即发布新品快讯
- **每日**: 网站更新「今日发现」板块
- **每周**: Newsletter 汇总本周最佳
- **每月**: 深度评测月度 Top 10

### 4.3 反馈闭环

```
用户行为追踪 → 内容效果分析 → 筛选算法优化 → 更精准的内容

指标:
- 点击率 (CTR)
- 页面停留时间
- 工具官网跳转率
- 用户评分/评论
```

## 五、竞品差异化

### 5.1 现有导航站的问题

| 问题 | 我们的解法 |
|------|-----------|
| 内容陈旧，更新慢 | 24小时发现机制 |
| 同质化严重 | 独家筛选算法，只选真正有创新的 |
| 只有列表，没有观点 | 每篇都有「是否值得跟进」的判断 |
| 缺乏社区反馈 | 实时聚合社区讨论热度 |

### 5.2 核心壁垒

1. **速度**: 比竞品早 24-48 小时发现新工具
2. **筛选**: 不是全量收录，只选真正有价值的
3. **观点**: 不只是信息，还有「我们的判断」
4. **数据**: 积累的热度预测模型，提前发现潜力股
