# AI Tools Navigator - 落地架构设计框架

## 调研目标
基于三模型（GLM5 + Kimi + MiniMax）并行调研：
1. 竞品技术架构分析
2. 完整系统架构设计
3. MVP最小可行方案
4. 自动化与成本优化

---

## 初步发现

### 竞品概览
**Futurepedia**:
- 4000+ 工具，50万+ 用户
- 不仅是导航站，还有 Skill Leap 教育平台
- 14天AI训练营等产品线
- 收入来源多元化

**TheresAnAIForThat**:
- 按用例/任务分类（非常细致）
- 大量更新日志（工具持续跟踪）
- 强大的搜索和筛选功能

### 核心洞察
1. **导航站只是入口**，真正的价值在教育、社区、数据
2. **内容更新频率**是核心竞争力
3. **分类方式**决定用户体验（按工具 vs 按任务）

---

## 架构设计原则

### 1. 渐进式架构
```
Phase 1: 静态站点 (Markdown/JSON)
Phase 2: 动态站点 (Next.js + 数据库)
Phase 3: 平台化 (用户系统、社区、API)
```

### 2. 自动化优先
- 工具信息自动抓取
- 内容自动生成（AI辅助）
- 更新自动检测

### 3. 成本敏感
- 充分利用免费额度
- 静态化减少服务器成本
- 外包图片存储到CDN
